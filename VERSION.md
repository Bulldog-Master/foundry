# Foundry Version

This file describes the current maturity state of Foundry. It is not software semantic versioning.

## Current generation: Foundry 0

Foundry 0 is the minimum permanent foundation from which Foundry evolves through real engineering work.

### Operating characteristics

- **One founder.** The founder is the sole ratifying authority.
- **One product.** Daang Remote is the only product currently developed under Foundry.
- **Products first.** Foundry exists to serve product work.
- **Minimal process.** Only the Constitution, the templates, and this version declaration exist. No workflows, CI, evaluation framework, governance handbook, memory system, or application scaffolding are in scope for this phase.
- **Reality driven.** Operational knowledge (ADRs, lessons, standards) becomes authoritative only when supported by evidence from real engineering work.
- **Human approval before merge.** No change lands without explicit founder approval.
- **Replaceable intelligence.** All work is authored so that any human or AI contributor can be substituted later without loss of continuity.

### When a new generation begins

A new Foundry generation is declared only when operating reality has materially changed — for example, when Foundry supports more than one product, when the founder is no longer the sole operator, or when the current operating model has demonstrably stopped fitting the work.

A new generation is not declared automatically upon the first ADR, the first lesson, or the first product milestone. Growth of content inside Foundry is expected; it does not by itself constitute a generational change.
